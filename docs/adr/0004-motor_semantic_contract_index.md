# ADR-0004 电机统一模型语义合同编号体系

## 背景 (Context)
- 统一模型需要长期演进，若没有合同编号，语义会在讨论与实现中漂移。
- 代码、测试、文档之间需要可回链的稳定锚点。

## 考虑过的方案 (Options)
- 方案 A：不设编号，仅靠自然语言文档维护语义。
- 方案 B：按模块临时编号，缺少统一规则。
- 方案 C：建立统一编号体系并强制回链。

## 最终决策 (Decision)
- 采用方案 C。
- 统一编号格式：`MM-<domain>-<nnn>`。
- 子域包含：`BUS/PROTO/CORE/CMD/STATE/SCH/SAFE/FLT/PARAM/DIAG/ENG/CAP`。
- 语义规则：合同编号发布后不得复用语义；新增合同必须附验证点。

## 影响 (Consequences)
- 评审与测试必须引用合同编号，禁止“只定义不验收”。
- 旧语义升级需新增编号或明确版本关系，不得静默改义。
- 统一模型相关文档的引用优先对齐 ADR 索引。
