# ADR-0001 OSAL 与 SYNC 语义边界

## 背景 (Context)
- 在并发、时间与同步机制设计中，`OSAL` 与 `SYNC` 的职责边界长期混用。
- 若不冻结边界，后续会出现“上层依赖未承诺语义”“平台差异泄漏到业务层”的问题。
- 该决策需成为跨模块统一约束，并与总分层规范保持一致。

## 考虑过的方案 (Options)
- 方案 A：由 `OSAL` 同时承载机制语义与协作语义。
- 方案 B：`OSAL` 仅承载机制语义合同，`SYNC` 承载协作语义合同。
- 方案 C：维持现状，按模块临时约定。

## 最终决策 (Decision)
- 采用方案 B。
- `OSAL` 负责跨平台稳定的机制语义合同：API 存在性、阻塞/超时语义、并发可见性、ISR 可用性、单调时间语义。
- `SYNC` 负责基于 `OSAL` 构建稳定协作语义：完成量、等待/触发语义、多等待者行为、状态迁移语义。
- 未写入合同的行为一律视为“明确不承诺”。

## 影响 (Consequences)
- `OSAL` 的接口演进必须保证机制语义不回退。
- `SYNC` 需要参考实现和一致性测试，禁止平台私有实现改变语义。
- 业务层不得依赖 `OSAL` 未承诺行为（公平性、唤醒顺序、性能特征）。
- 相关规范引用路径统一为 [`docs/architecture/layer_dependency_spec.md`](../architecture/layer_dependency_spec.md)。
