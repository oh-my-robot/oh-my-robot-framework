# ADR-0008 守护服务语义冻结（C3）

## 背景 (Context)
- 框架需要统一监测驱动、线程、服务等实体在线状态。
- 若各模块自行定义在线判定与恢复策略，会形成双事实源与不可审计行为。

## 考虑过的方案 (Options)
- 方案 A：继续由各模块维护局部在线判定。
- 方案 B：冻结统一守护语义（实体、租约、状态、事件），恢复策略留在业务层。
- 方案 C：在守护核心内直接内置自动恢复/重启策略。

## 最终决策 (Decision)
- 采用方案 B。
- 守护核心冻结统一语义：`entity_id` 主键、lease/heartbeat、ONLINE/OFFLINE 状态、边沿事件回调。
- 守护核心只输出事实，不承诺业务恢复动作。
- 回调执行与检测循环解耦，ISR 路径仅允许最小提交动作。

## 影响 (Consequences)
- 统一在线语义可跨模块复用并可审计。
- 恢复策略归属 `systems` 层，避免守护核心语义膨胀。
- 实现可优先复用 OSAL 单调时钟与 async 抽象，不新增重复接口。
