# 需求分册：基础数据结构（R1）

## 1. 目标

本分册只定义“必须补充哪些数据结构能力”，不做技术选型。  
所有需求均由 `event/msg/sync/async` 已冻结需求反推。

## 2. 上游需求映射

1. DS-01（队列）来源：EVT-03、MSG-01、MSG-02、ASY-01、ASY-04
2. DS-02（索引）来源：EVT-02、MSG-01、MSG-03、ASY-02
3. DS-03（资源池）来源：EVT-01、EVT-02、MSG-05、ASY-04

## 3. 需求清单（严格字段）

## DS-01 队列能力需求

1. 来源：EVT-03、MSG-01、MSG-02、ASY-01、ASY-04。
2. 目标场景：承载异步分发、消息排队与背压处理。
3. 输入/输出语义：输入为入队元素；输出为出队元素或显式错误。
4. 成功判据：容量边界可配置且行为确定。
5. 失败语义：队列满、空队列、非法参数显式返回。
6. 边界与不做项：本阶段不定义 ring/list/heap 等具体实现选型。
7. 下层能力诉求：无（本域最低层能力）。
8. 可验证条件：顺序一致性、边界行为、过载行为可复现验证。

## DS-02 索引能力需求

1. 来源：EVT-02、MSG-01、MSG-03、ASY-02。
2. 目标场景：事件绑定关系、订阅关系、请求关联查找。
3. 输入/输出语义：输入为键和值关联；输出为查询/更新/删除结果。
4. 成功判据：键空间冲突行为可定义且可验证。
5. 失败语义：键不存在、冲突不可解、容量不足显式返回。
6. 边界与不做项：本阶段不定义 hash/tree 等具体选型。
7. 下层能力诉求：无（本域最低层能力）。
8. 可验证条件：插入/查找/删除一致性与冲突场景可验证。

## DS-03 资源池能力需求

1. 来源：EVT-01、EVT-02、MSG-05、ASY-04。
2. 目标场景：固定容量资源分配（句柄、槽位、上下文对象）。
3. 输入/输出语义：输入为申请/释放操作；输出为资源句柄或显式错误。
4. 成功判据：资源上界固定且可审计，释放后可复用。
5. 失败语义：耗尽、重复释放、非法句柄显式返回。
6. 边界与不做项：本阶段不定义 bitmap/freelist 等实现选型。
7. 下层能力诉求：无（本域最低层能力）。
8. 可验证条件：耗尽恢复、重复释放、并发申请边界场景可验证。

## 4. 不做项

1. 不定义具体数据结构算法。
2. 不定义并发实现细节。
3. 不定义内存布局与对齐策略。
