# 电机统一模型研发规划与实现优先级（v1.0）

------

## 顶层执行流程（强制）

后续电机统一模型工作按以下固定顺序执行：

1. **确认需求边界**：目标、范围、非目标、验收标准。
2. **冻结语义合同**：先更新合同索引，再进入实现动作。
3. **分阶段实施**：每阶段都包含输入/输出/门禁/风险/回退。
4. **证据化验证**：每阶段必须提供可回溯验证证据。
5. **进度留痕**：阶段结论必须回填到进度说明文档。

------

## 总体指导思想（非常重要）

> **先锁完整语义合同，再做实现适配；先保旧接口外观，再收敛内部模型。**

因此：

- 当前优先级不是一次性替换全部接口，而是建立“合同驱动的统一模型”。
- `dji_*`、`p1010b_*` 先保留签名，通过内部映射过渡。
- 每阶段门禁聚焦“语义正确性优先于性能优化”。

------

## Phase 0：合同与边界冻结（P0）

### 目标

冻结完整架构考虑点的合同编号与边界，不留“实现再决定”的空白区。

### 必做事项

1. 冻结合同域：
   - `MM-BUS/MM-PROTO/MM-CORE/MM-CMD/MM-STATE/MM-SCH/MM-SAFE/MM-FLT/MM-PARAM/MM-DIAG/MM-ENG/MM-CAP`
2. 冻结层级边界：
   - Core/Transport/Vendor Adapter/Application 责任与禁止项
3. 冻结命名规则：
   - 公共类型不使用版本号后缀

### 输出物

- 语义合同索引
- 顶层设计规范
- 责任边界说明

### 验收门禁

1. 合同域覆盖完整且编号无冲突。
2. 边界文档与合同文档互相引用闭环。
3. 无 `...V1` 类型命名残留。

### 风险与回退

- 风险：合同条目过于抽象导致不可执行。
- 回退：补充“输入约束/输出保证/验证点”后再冻结。

------

## Phase 1：统一模型最小可运行壳（P0）

### 目标

建立覆盖控制/状态/故障/调度的 Core 壳体，并保留旧接口外观。

### 必做事项

1. 定义统一公共类型：
   - `MotorCommand`
   - `MotorSnapshot`
   - `MotorState`
   - `MotorFaultInfo`
   - `MotorCapabilityProfile`
2. 定义统一最小接口：
   - `set_command`
   - `process_rx`
   - `flush_tx`
   - `tick`
   - `get_snapshot`
   - `get_fault`
3. 建立 Transport 抽象壳与 Adapter 接口壳。

### 输出物

- Core 壳头文件与最小实现
- 编译通过证据

### 验收门禁

1. 公共头不泄露 vendor 协议字段。
2. 旧接口签名未破坏。
3. Core 可承载旧接口映射。

### 风险与回退

- 风险：壳层字段定义过度或不足。
- 回退：按合同索引最小字段集收敛。

------

## Phase 2：双样板适配（DJI + P1010B）（P0/P1）

### 目标

用两类差异最大的 vendor 路径验证统一模型可行性。

### 必做事项

1. `DJI` 适配：
   - 组帧槽位语义映射
   - 边沿故障回调语义映射
2. `P1010B` 适配：
   - 模式驱动语义映射
   - 状态守卫与超时语义映射
3. 生成两个接口族的 `compat_note`。

### 输出物

- `dji -> unified_model` 映射层
- `p1010b -> unified_model` 映射层
- 差异留痕文档

### 验收门禁

1. 旧样例入口不改调用即可构建通过。
2. 核心控制与故障路径无语义弱化。
3. 不支持能力明确失败。

### 风险与回退

- 风险：映射引入隐式行为改变。
- 回退：按合同编号逐项回归并局部回退映射点。

------

## Phase 3：达妙适配与参数/诊断收敛（P1）

### 目标

补齐达妙契约缺口，完成参数语义与诊断语义统一收敛。

### 必做事项

1. 补齐 `damiao` 头文件契约（摆脱仅 PDF 事实依赖）。
2. 完成 `damiao -> unified_model` 映射层。
3. 实现参数门禁、持久化语义与诊断计数最小闭环。

### 输出物

- `damiao` 契约头文件
- 达妙映射实现
- 参数/诊断闭环证据

### 验收门禁

1. 达妙路径最小控制流程可运行。
2. 参数门禁语义与合同一致。
3. 诊断计数与事件可观测。

### 风险与回退

- 风险：文档事实与真实固件行为不一致。
- 回退：保留 `vendor_rom_code` 并标注成熟度降级。

------

## Phase 4：回归收敛与发布留痕（P1）

### 目标

完成合同一致性、跨 vendor 一致性与文档一致性收敛。

### 必做事项

1. 回归控制/状态/故障/参数/诊断关键路径。
2. 执行合同覆盖率审计。
3. 回填进度文档阶段结论与未完成项。

### 输出物

- 回归报告
- 合同覆盖率报告
- 发布留痕记录

### 验收门禁

1. 关键路径无回归退化。
2. 合同编号均有实现或测试证据。
3. 进度说明无模糊结论。

### 风险与回退

- 风险：跨 vendor 语义仍有冲突。
- 回退：冻结冲突点，新增合同条目后继续推进。

------

## 默认优先级

1. `P0`：合同冻结 + 最小壳 + 双样板适配
2. `P1`：达妙适配 + 参数诊断收敛 + 回归
3. `P2`：性能优化与扩展能力（不影响核心合同）

------

## 明确不在当前规划范围

- ECAT 接入相关设计与实现

3. 新语义必须先登记合同编号再开发。
