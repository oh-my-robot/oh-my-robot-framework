# 需求分册：同步能力（R1）

## 1. 目标

同步能力只负责并发语义，不承载消息或事件业务语义。  
本分册由 `event/msg` 需求反推，定义同步层必须提供的脚手架能力。

## 2. 上游需求映射

1. EVT-02 -> SYN-03
2. EVT-04 -> SYN-01
3. EVT-05 -> SYN-01
4. MSG-03 -> SYN-03
5. MSG-04 -> SYN-01、SYN-03
6. MSG-05 -> SYN-01

## 3. 需求清单（严格字段）

## SYN-01 单次完成通知能力

1. 来源：EVT-04、EVT-05、MSG-04、MSG-05。
2. 目标场景：同步等待某个一次性事件完成。
3. 输入/输出语义：输入为等待对象与超时；输出为完成或超时。
4. 成功判据：等待/通知语义一致，重复通知行为确定。
5. 失败语义：非法句柄、超时、并发冲突显式返回。
6. 边界与不做项：本阶段不定义后端加速实现。
7. 下层能力诉求：OSAL sem/event 原语。
8. 可验证条件：先通知后等待、先等待后通知、超时三路径可验证。

## SYN-02 事件位等待语义

1. 来源：EVT-01、EVT-03。
2. 目标场景：线程等待一组事件位满足条件后继续执行。
3. 输入/输出语义：输入为等待掩码与模式；输出为命中掩码与状态。
4. 成功判据：wait-any / wait-all 行为可判定且一致。
5. 失败语义：非法掩码、上下文非法、超时返回显式错误。
6. 边界与不做项：本阶段不定义位图存储实现。
7. 下层能力诉求：OSAL event 原语。
8. 可验证条件：all/any/no-clear 组合场景可验证。

## SYN-03 多源等待聚合（waitset）

1. 来源：EVT-02、MSG-03、MSG-04。
2. 目标场景：单线程等待多个同步源（完成、事件、超时、取消）。
3. 输入/输出语义：输入为等待集合；输出为触发源标识与状态。
4. 成功判据：触发源识别唯一且超时/取消优先级语义确定。
5. 失败语义：无效集合、重复注册、非法移除返回显式错误。
6. 边界与不做项：本阶段不定义内部调度数据结构。
7. 下层能力诉求：SYN-01、SYN-02、OSAL time。
8. 可验证条件：多源竞态与超时取消组合测试可复现。

## SYN-04 统一超时与取消语义

1. 来源：MSG-03、MSG-04、EVT-03。
2. 目标场景：跨同步原语保持一致错误码与状态回收语义。
3. 输入/输出语义：输入为等待/取消操作；输出为统一状态码。
4. 成功判据：不同同步原语对同类失败返回一致类别。
5. 失败语义：取消后遗留状态可检测且可清理。
6. 边界与不做项：本阶段不定义取消传播算法。
7. 下层能力诉求：OSAL time、thread、event。
8. 可验证条件：超时与取消交错场景一致性可验证。

## 4. 不做项

1. 不定义同步 API 细节签名。
2. 不定义端口特化或加速后端选择。
3. 不定义业务层对象模型。
