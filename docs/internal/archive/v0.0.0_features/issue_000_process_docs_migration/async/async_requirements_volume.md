# 需求分册：异步能力（R1）

## 1. 目标

异步能力负责执行编排，不承载业务语义。  
本分册由 `event/msg` 需求反推，定义异步层必须提供的脚手架能力。

## 2. 上游需求映射

1. EVT-03 -> ASY-01、ASY-04
2. EVT-04 -> ASY-01、ASY-03
3. MSG-01 -> ASY-01
4. MSG-02 -> ASY-04
5. MSG-03 -> ASY-01、ASY-02
6. MSG-04 -> ASY-01
7. MSG-05 -> ASY-03

## 3. 需求清单（严格字段）

## ASY-01 工作队列执行能力

1. 来源：EVT-03、EVT-04、MSG-01、MSG-03、MSG-04。
2. 目标场景：任务提交后由异步执行上下文消费。
3. 输入/输出语义：输入为任务单元与上下文；输出为执行结果状态。
4. 成功判据：提交、排队、执行语义可观测且一致。
5. 失败语义：队列满、非法任务、生命周期非法显式返回。
6. 边界与不做项：本阶段不定义线程池调度算法。
7. 下层能力诉求：DS-01、SYN-04。
8. 可验证条件：正常/队列满/停止态提交三场景可验证。

## ASY-02 延时队列能力

1. 来源：MSG-03（超时控制）、EVT-03（延迟分发扩展）。
2. 目标场景：任务按截止时间触发，支持取消。
3. 输入/输出语义：输入为任务与deadline；输出为触发或取消结果。
4. 成功判据：延时触发语义与超时语义一致。
5. 失败语义：非法时间参数、重复取消、资源不足显式返回。
6. 边界与不做项：本阶段不定义具体时间组织结构。
7. 下层能力诉求：OSAL time、SYN-04、DS-02。
8. 可验证条件：准时触发、提前取消、超时未触发三场景可验证。

## ASY-03 生命周期治理

1. 来源：EVT-04、MSG-05。
2. 目标场景：异步执行器初始化、启动、停止、退出可控。
3. 输入/输出语义：输入为生命周期命令；输出为状态转换结果。
4. 成功判据：状态转换单向可审计，不出现悬挂任务泄漏。
5. 失败语义：非法状态迁移返回显式错误。
6. 边界与不做项：本阶段不定义内部状态机实现结构。
7. 下层能力诉求：SYN-01、SYN-04。
8. 可验证条件：状态迁移矩阵可全覆盖测试。

## ASY-04 背压与退化语义

1. 来源：EVT-03、MSG-02。
2. 目标场景：任务过载时系统有确定退化行为而非失控。
3. 输入/输出语义：输入为负载压力；输出为接收/拒绝/丢弃结果。
4. 成功判据：过载行为策略可配置且可观测。
5. 失败语义：容量不足返回显式错误，不允许静默失败。
6. 边界与不做项：本阶段不定义优先级抢占算法。
7. 下层能力诉求：DS-01、DS-03。
8. 可验证条件：高水位压测下行为可重复验证。

## 4. 不做项

1. 不定义执行器 API 签名。
2. 不定义线程模型与调度策略实现。
3. 不定义具体背压算法实现。
